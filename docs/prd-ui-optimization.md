# 产品需求文档 - UI 优化与主题增强

## 1. 需求背景

当前 eh-mp-formatter 第一版基础功能已实现，但存在以下问题：
- UI 设计缺乏专业感，不够美观
- 主题配色单一，不够丰富
- Markdown 渲染样式不完整（如列表符号未正确显示）
- 主题样式效果不够丰富

需要从专业 UI/UX 角度进行全面优化。

## 2. 功能列表

| ID | 功能名称 | 优先级 | 描述 |
|----|----------|--------|------|
| F1 | 首页 UI 重构 | P0 | 重新设计整体布局和视觉风格 |
| F2 | 主题配色增强 | P0 | 新增 10+ 套专业配色主题 |
| F3 | Markdown 样式修复 | P0 | 修复列表、引用等渲染问题 |
| F4 | 主题效果丰富化 | P1 | 添加更多视觉效果（阴影、圆角、动画） |
| F5 | 代码风格增强 | P1 | 新增更多代码高亮主题 |

## 3. 验收标准

### F1: 首页 UI 重构
- [ ] 整体布局更专业，符合现代设计规范
- [ ] 工具栏设计更美观，图标清晰
- [ ] 编辑器和预览区视觉层次分明
- [ ] 底部操作栏按钮设计统一

### F2: 主题配色增强
- [ ] 至少 10 套专业配色主题
- [ ] 每套主题配色协调、专业
- [ ] 主题切换流畅，预览即时更新

### F3: Markdown 样式修复
- [ ] 无序列表正确显示圆点
- [ ] 有序列表正确显示数字
- [ ] 引用块样式正确
- [ ] 代码块样式正确

### F4: 主题效果丰富化
- [ ] 卡片阴影效果
- [ ] 圆角统一
- [ ] 过渡动画流畅

### F5: 代码风格增强
- [ ] 至少 5 套代码高亮主题
- [ ] 支持一键切换

## 4. 技术方案

### 4.1 UI 设计原则

**整体风格**: 现代、简洁、专业

**设计规范**:
- 主色调：采用低饱和度配色
- 字体：系统字体栈 + 代码字体
- 间距：8px 基础单位
- 圆角：4px/8px/12px 三档
- 阴影：轻量级阴影增加层次感

### 4.3 Markdown 样式规范

```css
/* 无序列表 */
ul { list-style-type: disc; padding-left: 1.5em; }
ul li { margin: 0.25em 0; }

/* 有序列表 */
ol { list-style-type: decimal; padding-left: 1.5em; }
ol li { margin: 0.25em 0; }

/* 引用块 */
blockquote {
  border-left: 4px solid var(--theme-color);
  background: var(--theme-bg);
  padding: 1em 1.5em;
  border-radius: 0 8px 8px 0;
}

/* 代码块 */
pre {
  background: #1e1e1e;
  border-radius: 8px;
  padding: 1em;
  overflow-x: auto;
}
```

### 4.4 关键流程

## 5. 风险点

| 风险 | 影响 | 应对方案 |
|------|------|----------|
| 主题过多影响性能 | 中 | 按需加载主题 CSS |
| 样式冲突 | 中 | 使用 CSS 变量隔离 |
| 兼容性问题 | 低 | 测试主流浏览器 |
