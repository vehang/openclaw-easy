# 2026-04-16 工作记录

## 与用户的产品讨论

### 讨论主题：个人碎片知识管理工具

**用户痛点：**
- 命令/笔记散落在浏览器、微信、备忘录各处
- 来回复制粘贴，繁琐且找不到
- 随手记久了杂乱无章，难以检索
- 希望便携、轻量、跨平台

**共识方案方向：**
- 产品形态：全局快捷面板（轻量、跨平台，类似 Raycast/Albert）
- 核心机制：极简 Capture（< 2秒记录）+ 智能 Retrieve（模糊搜索 + 上下文推荐）
- 关键设计：用"搜索代替分类"，系统渐进式学习用户习惯
- 不只是命令：各种日后用得上的内容都可以记录

**待确认细节：**
- 主力系统（Mac/Linux/Win）
- 数据存储偏好（本地 vs 云同步）
- 是否有代码片段特殊需求

## OpenClaw 磁盘占用分析

帮用户分析了 OpenClaw 相关目录占用：
- `~/.openclaw`：约 3.9 GB（含 workspace/extensions/data/tools 等）
- `/tmp/openclaw/`：约 70 MB（日志，每天约 30-40MB）
- `~/.npm`：357 MB

用户目的是把 OpenClaw 迁移到大容量磁盘挂载。
