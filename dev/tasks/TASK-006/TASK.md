# TASK-006: 全部工具页面UI优化

## 任务类型
UI优化 + Bug修复

## 问题清单

### 问题1: TabBar 右侧按钮错位
- **现象**: 手机端 TabBar 右侧"查看全部"按钮挤错位了滚动按钮
- **原因**: "查看全部"按钮与滚动按钮布局冲突
- **方案**: 重新设计布局，考虑将"查看全部"按钮移到其他位置或调整层级

### 问题2: 详情列表顶部菜单信息不美观
- **现象**: 顶部菜单信息显示很丑，滚动时消失
- **方案**: 重新设计顶部区域样式，优化滚动体验

### 问题3: 全部工具页面整体不美观
- **现象**: 中间区域结合Header+Footer整体不协调
- **方案**: 美化中间展示区域，优化整体布局

### 问题4: 自动slug太长不好看
- **现象**: Base64编码的slug太长，如 `/tools/c-JUU0JUJDJTk4...`
- **方案**: 改用菜单ID作为slug，更简洁直观
- **实现**: 需要确认Category是否有ID字段，如果没有需要添加或使用其他唯一标识

## 验收标准

- [ ] 顶部菜单信息美观
- [ ] 页面整体布局美观协调
