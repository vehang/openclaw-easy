# TASK-005: 全部工具页面彻底优化

## 任务类型
Bug修复 + 设计优化

## 问题诊断

### 问题1: 部分菜单没有"查看全部"按钮
- **原因**: `CATEGORY_SLUG_MAP` 只定义了4个分类的slug映射
- **影响**: 其他分类的 `getCategorySlug()` 返回 `null`，按钮不显示
- **表现**: 不一致的UI体验

### 问题2: slug映射设计不合理
- **问题**: 手动维护映射表，容易遗漏新增分类
- **改进**: 应自动生成slug，无需手动维护

### 问题3: 路由结构不清晰
- **现状**: `/tools/[slug]/all` 动态路由
- **问题**: slug映射不全导致404

## 优化方案

### 方案1: 自动slug生成
- 移除手动映射表
- 使用拼音或编码自动生成slug
- 所有分类都有对应的"查看全部"入口

### 方案2: 使用分类ID代替slug
- 后端已有分类ID，直接使用ID
- `/tools/[categoryId]/all`
- 更稳定，无需维护映射

### 方案3: 统一入口
- 单一"全部工具"页面，显示所有分类
- 每个分类卡片可展开查看全部工具
- 更清晰的用户体验

## 验收标准
- [ ] 所有分类都有"查看全部"入口
- [ ] 入口行为一致
- [ ] 页面级浏览器测试验证
- [ ] 构建成功
- [ ] 部署成功
