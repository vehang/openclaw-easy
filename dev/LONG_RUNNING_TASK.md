# 长时间任务执行规范（v2）

> 本规范适用于所有机器人（小码、犊子、牛马）
> 
> **⚠️ 核心要求：必须主动汇报进度和完成通知！**

---

## 🔴 核心要求

### 1. 必须按时汇报（每10分钟）
### 2. 任务完成必须主动通知
### 3. 阶段切换必须主动通知
### 4. **必须开启 tmux 日志监控**
### 5. **任务结束后必须总结工作内容**

**违规后果**：用户无法信任任务执行，影响使用体验

---

## 📋 tmux 日志监控（强制执行）

### 任务启动时

```bash
# 1. 创建 tmux 会话
tmux new-session -d -s {会话名} -c {项目目录}

# 2. ⭐ 开启日志监控
tmux pipe-pane -o -t {会话名} "cat >> /tmp/task-{任务ID}.log"

# 3. ⭐ 启动飞书监控脚本（后台运行）
./scripts/task-monitor-feishu.sh {任务ID} {会话名} /tmp/task-{任务ID}.log {账号} &

# 4. 启动 Claude Code
tmux send-keys -t {会话名} 'claude --dangerously-skip-permissions --model glm-5' Enter
```

### 监控脚本功能

**自动执行**：
- 每10分钟读取 tmux 日志
- 通过飞书 API 发送进度汇报
- 检测任务完成 → 发送汇总报告
- 任务完成 → 脚本自动退出

**不需要用户触发！**

### 每10分钟汇报

```bash
# 读取日志最新内容
tail -50 /tmp/task-{任务ID}.log

# 使用 message 发送汇报
message({
  action: "send",
  message: "📊 进度汇报\n\n" + 日志内容摘要
})
```

### 任务结束时

```bash
# 1. 读取完整日志
cat /tmp/task-{任务ID}.log

# 2. 总结工作内容
- 完成了什么
- 创建了哪些文件
- 修改了哪些文件
- 遇到了什么问题
- 如何解决的

# 3. 使用 message 发送总结报告
message({
  action: "send", 
  message: "✅ 任务完成！\n\n" + 完整总结
})
```

## ⚠️ 重要限制说明

### 我无法主动定时触发

**原因**: AI 是被动的，需要用户消息或 heartbeat 触发

**解决方案**:
1. **用户主动询问** → 我立即检查并汇报
2. **Heartbeat 触发** → 配置定时 heartbeat（推荐）
3. **关键节点主动汇报** → 阶段完成/任务完成时立即发送 message

### Heartbeat 配置

在 `HEARTBEAT.md` 中添加：

```markdown
# Heartbeat 任务

## 长时间任务监控
如果有 running 状态的任务，检查进度并发送 message 汇报。
```
