# 2026-04-04 Memory

## OpenClaw Docker Root 版本构建

### 背景
用户询问为什么 OpenClaw Docker 镜像使用 `node` 用户而不是 `root`，分析后认为作为智能体工具应该有高权限。

### 分析结论
- 官方镜像用 `node` 用户是基于安全考虑（容器逃逸防护）
- 但 OpenClaw 作为智能体需要安装软件、执行命令，权限限制会影响使用体验
- 建议提供选择权或默认 root 版本

### 执行的工作
1. 克隆项目: `https://github.com/justlovemaki/openclaw-docker-cn-im`
2. 修改 Dockerfile 创建纯 root 版本：
   - 删除所有 `USER node` 和 `USER root` 语句
   - 将所有 `/home/node` 路径改为 `/root`
   - 删除所有 `chown -R node:node` 操作
   - 删除 `gosu node` 降权逻辑
3. 修改 init.sh：
   - 移除 gosu 降权
   - 所有路径改为 /root
4. 添加国内镜像加速：
   - apt: `http://mirrors.aliyun.com`
   - npm: `https://registry.npmmirror.com`
   - pip: `https://pypi.tuna.tsinghua.edu.cn/simple`
   - Homebrew: `https://mirrors.tuna.tsinghua.edu.cn`

### 构建过程中的问题

#### 1. 磁盘空间不足
- 症状: `ENOSPC: no space left on device`
- 解决: `docker system prune -af --volumes` 清理了 26.62GB

#### 2. 构建被 SIGTERM 终止
- 症状: 构建运行约 50-60 分钟后被终止
- 可能原因: OpenClaw 会话超时限制
- 解决方案: 改为串行构建，一个一个版本构建

#### 3. 并行构建资源争抢
- 症状: 两个版本同时构建时容易失败
- 解决: 改为串行构建

### 国内镜像加速配置 (已添加到 Dockerfile)

```dockerfile
# apt 阿里云镜像
RUN sed -i 's|http://deb.debian.org|http://mirrors.aliyun.com|g' /etc/apt/sources.list.d/debian.sources

# npm 淘宝镜像
npm config set registry https://registry.npmmirror.com

# pip 清华镜像
/usr/local/bin/python3 -m pip install --no-cache-dir -i https://pypi.tuna.tsinghua.edu.cn/simple websockify

# Homebrew 清华镜像
git clone --depth 1 https://mirrors.tuna.tsinghua.edu.cn/git/homebrew/brew.git
```

加速效果: apt 下载 487 MB 从原来很慢变成 1-2 分钟完成

### x86/ARM64 兼容性
- 基础镜像支持多架构
- apt/npm 包自动适配架构
- 无需额外修改即可在两种架构上构建

### ⭐ 优化方案：基础镜像 + 版本镜像 (推荐)

为了避免每次都从头构建，创建了分层架构：

```
openclaw-base:latest (基础镜像 ~3.5GB)
    ├── 系统依赖 (chromium, ffmpeg, docker.io, build-essential)
    ├── bun, uv, qmd
    ├── Playwright Chromium (~170MB)
    ├── Homebrew
    └── 插件种子
    │
    ├── openclaw:2026.4.1-root (只安装 openclaw@2026.4.1)
    ├── openclaw:2026.3.24-root (只安装 openclaw@2026.3.24)
    └── openclaw:2026.3.13-root (只安装 openclaw@2026.3.13)
```

#### 新增文件

| 文件 | 说明 |
|-----|------|
| `Dockerfile.base` | 基础镜像 (约 15-20 分钟构建一次) |
| `Dockerfile.version-template` | 版本镜像模板 (只需 2-3 分钟) |
| `build.sh` | 便捷构建脚本 |

#### 构建效率对比

| 方案 | 构建时间 | 空间占用 |
|-----|---------|---------|
| 之前 (每个版本独立构建) | 15-20 分钟 × 3 = 45-60 分钟 | ~12.5GB |
| 优化后 (基础镜像 + 版本镜像) | 20 分钟 + 3 分钟 × 3 = 29 分钟 | ~5GB (共享基础层) |

#### 构建命令

```bash
cd ~/.openclaw/workspace/dev/openclaw-docker-cn-im

# 构建基础镜像 (只需一次)
./build.sh base

# 构建指定版本 (快速!)
./build.sh version 2026.4.1
./build.sh version 2026.3.24
./build.sh version 2026.3.13

# 或一次性构建所有
./build.sh all
```

### 使用方法

```bash
# 运行容器（需要配置 API_KEY）
docker run -d \
  -e API_KEY=your-api-key \
  -e BASE_URL=https://api.openai.com/v1 \
  -e MODEL_ID=gpt-4o \
  -p 18789:18789 \
  -v openclaw-data:/root/.openclaw \
  openclaw:2026.4.1-root

# 进入容器（root 权限）
docker exec -it <container_id> bash

# 验证运行用户
docker run --rm --entrypoint "" openclaw:2026.4.1-root whoami
# 输出: root
```
