### 4. login/setup 重定向死循环修复

**根本原因**: 
- `app.js` 中的 `checkAuthStatus()` 前端检查
- `server.js` 中间件的后端重定向
- 两者同时工作，导致循环

**修复方案**:
1. 移除 `app.js` 中的前端状态检查
2. 后端中间件统一处理页面重定向
3. 给重定向添加 `Cache-Control: no-store`

**中间件逻辑**:
```
login.html: 密码未设置→setup，已设置→停留
setup.html: 密码已设置→login，未设置→停留
其他页面: 未设置密码→setup，未登录→login
```

## 关键经验

### 1. 前端/后端重定向不要同时做

职责划分：
- 页面重定向：后端中间件统一处理
- API 返回：前端只处理 Toast/表单

### 2. Express 路由顺序很重要

所有 API 路由必须在：
- `express.static` 之后
- 404 处理之前

### 3. 外部 script 标签要正确闭合

```html
<script src="xxx.js"></script>  <!-- 正确 -->
<script src="xxx.js">           <!-- 错误，会吞掉后面的内容 -->
```

### 4. 浏览器会缓存 302 重定向

所有重定向添加：
```javascript
res.set('Cache-Control', 'no-store').redirect('/xxx')
```

### 9. 微信二维码登录 - APP客户端方案（讨论）

**需求背景**：
- 外部端已有 SSE 实时推送登录过程
- APP客户端需要独立的接口获取二维码状态

**接口设计**：

| 接口 | 方法 | 说明 |
|------|------|------|
| `/api/weixin/qr/start` | POST | 启动异步任务，自动停止旧任务 |
| `/api/weixin/qr/status` | GET | 获取实时状态 |

**状态定义**：

| status | 说明 |
|--------|------|
| `loading` | 正在加载二维码 |
| `waiting` | 二维码已生成，等待扫码 |
| `success` | 登录成功 |
| `timeout` | 二维码超时 |
| `error` | 出错 |

**文件存储**：
- `.weixin-qr-state.json` - 二维码状态文件
- `.weixin-bound` - 绑定状态文件（现有）

**进程管理**：
- 全局变量 `currentWeixinTask` 控制进程
- 与现有 `/api/weixin/login` SSE 接口共享
- 只允许一个进程运行，互斥控制

**二维码链接解析**（正则）：
```javascript
/https?:\/\/liteapp\.weixin\.qq\.com\/q\/[^\s\]\n]+/g
```

**APP调用流程**：
```
POST /start → 返回 taskId → 后台异步执行
循环 GET /status → 获取最新状态
status = success → 登录成功
```

**状态**：方案已讨论完成，待实现
