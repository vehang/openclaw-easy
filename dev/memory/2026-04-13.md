## openclaw-easy 安全更新机制重构

### 背景
原来的更新流程是原地替换文件 + npm install + 重启，存在以下问题：
- 更新过程中 npm install 可能失败，导致 18780 服务起不来
- Node.js 进程自己重启自己，存在竞态条件
- Web更新路径没有回滚保护
- 三条更新路径（Web/自动/手动 shell）各自实现，逻辑不一致

### 新方案：同级目录 + 测试启动 + mv 切换
1. 下载更新包，解压到 `/app/openclaw-easy-new/`（同级目录）
2. 从当前运行目录 `cp -a node_modules` 到新目录
3. 在新目录执行 `npm install --production`
4. 在 18781 端口测试启动新版本，`curl /api/status` 验证
5. 测试通过 → `supervisorctl stop` → `mv 旧目录当备份` → `mv 新目录改名` → `supervisorctl start`
6. 切换后验证启动，失败自动回滚

### 优势
- 18780 在整个更新过程中不受影响，只有最后 mv 切换的毫秒级中断
- 测试失败 = 不切换 = 无需回滚
- 天然安全，不会出现更新后起不来的情况

### 统一更新路径
三条更新路径全部走 `docker/update.sh update <url>`：
- Web 一键更新：`POST /update` → `performUpdate()` → shell 脚本
- 自动更新：`tasks/auto-update.js` → `performUpdate()` → shell 脚本
- 手动更新：`docker/update.sh update <url>` → shell 脚本
