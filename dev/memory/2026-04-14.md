## 一、App API 认证体系（openclaw-easy）

### 背景
feature/modular-refactor 分支已有 `/api/auth/token` + `authMiddleware`，但：
- 5 个 App 接口没加中间件，实际未生效
- Token 纯内存，重启即丢失
- 有效期只有 24h，App 端体验差
- 无开关控制

### 实施内容（基于 feature/modular-refactor 分支）

新增/修改文件：

| 文件 | 改动 |
|------|------|
| `constants/config.js` | +APP_TOKENS_FILE, APP_TOKEN_EXPIRE_TIME; PORT 环境变量覆盖 |
| `state/app-tokens.js` | **新增**：持久化 token 管理（内存+文件） |
| `state/index.js` | 注册 app-tokens 模块 |
| `middleware/auth.js` | 新增 appAuthMiddleware（5种来源提取 + 开关控制） |
| `routes/auth.js` | 改造 /api/auth/token 用持久化 token（7天有效） |
| `routes/config.js` | config/simple POST/GET 加 appAuthMiddleware |
| `routes/system.js` | /fix 加 appAuthMiddleware |
| `routes/weixin.js` | qr/start, qr/status 加 appAuthMiddleware |
| `server.js` | 启动时 loadAppTokens() |
| `version.json` | 升级到 1.0.0.6, 新增 appAuthRequired: true |

### 认证流程
```
App 发送 token+barCode → /api/auth/token → 远程验证 → 生成/复用 accessToken（7天有效，持久化）
→ 后续请求携带 accessToken → appAuthMiddleware 校验（5种来源：Header/Query/Body/Cookie）
```

- `docs/API-AUTH-DESIGN.md` — App API 认证方案 & 实施明细

## 二、更新机制重构（openclaw-easy）

### 旧方案问题
- Node 进程内做文件操作+npm install+重启（进程自更新有风险）
- JS(370行) + Shell(527行) 两套重复逻辑
- 原地覆盖无安全窗口

### 新方案：同级目录 + 测试启动 + mv 切换

8 步流程：准备→备份→下载→解压到新目录→cp -a node_modules→npm install→测试启动(18781)→mv切换

核心优势：
- 18780 服务在整个更新过程中正常运行
- 测试失败 = 不切换 = 不需要回滚
- mv 切换只有毫秒级中断
- 三条路径统一走 shell 脚本

### 修改文件

| 文件 | 改动 |
|------|------|
| `docker/update.sh` | 完全重写（排除法→同级目录法，8步流程） |
| `routes/update.js` | 大幅简化（370→169行，删除文件操作逻辑） |
| `docker/start-check.sh` | 增加 bak 目录检测回滚 |
| `constants/config.js` | PORT 环境变量覆盖 |

### 文档
- `docs/UPDATE-REDESIGN.md` — 更新机制重构方案

## 五、工具调研

### oh-my-openagent (OmO)
- OpenCode/Claude Code 的增强插件
- 多模型协作、Hashline编辑、后台Agent
- 需要OpenCode+多个AI订阅，与OpenClaw不兼容
- 结论：不适合当前环境

### awesome-design-md
- 66+ 品牌的 DESIGN.md 设计规范文件集合
- 丢到项目里，AI Agent 自动生成对应风格的 UI
- 可用于 baoboxs-nav 前端开发
- 推荐风格：Vercel/Linear/Notion

### Superpowers (obra/superpowers)
- AI 编码 Agent 的 Skills 框架 + 开发方法论
- 支持 brainstorming→计划→TDD→subagent→code review 完整流程
- 是 Claude Code/Cursor 插件，不是 OpenClaw skill
- 适合用 Claude Code 做开发时装

### Hermes Agent (NousResearch)
- 自学习 AI Agent，与 OpenClaw 同类竞品
- 特色：自动创建/改进 Skill、跨会话记忆、用户建模
- 支持 `hermes claw migrate` 从 OpenClaw 迁移
- 创建了 Docker 部署项目 `projects/hermes-docker/`
- 已完成构建并启动（2026-04-17）
