## openclaw-nim-yx-auth 插件

### 版本升级 0.3.0 → 1.0.6
- 依赖 `openclaw-nim` 从 ^0.3.0 升级到 ^1.0.6，移除 `nim-web-sdk-ng`（已内置 @yxim/nim-bot）
- 注入格式从扁平 `channels.nim = {...}` 改为 `channels.nim.instances[0] = {...}`
- 设置 `ctx.accountId = appKey:account` 适配 1.0.6 多实例
- auth.ts、types.ts 不变，只改 channel.ts
- commit: 303b415（main）

### npm 发布
- npm 账号：tilldream
- 包名：openclaw-nim-yx-auth
- **1.0.6**（latest）— openclaw ≥2026.4.5，openclaw-nim 1.0.6，instances 格式
- **0.3.0**（legacy tag）— openclaw ≥2026.1.29，openclaw-nim 0.3.0，扁平格式
- 基于 commit 3ff57d1 发布了 0.3.0
- npm token 在 ~/.npmrc（已加入 .gitignore）
- 发布命令：`npm publish --registry https://registry.npmjs.org`

### 日志系统
- 新增 logger.ts，同时输出 console 和文件（/var/log/supervisor/nim-yx-auth.log）
- index.ts、channel.ts、auth.ts 全部改用 logger
- 8 步 startAccount 日志，任何步骤失败有 ERROR 级别日志
- commit: 43094e8（main）

### 1.0.6 兼容性
- **openclaw 2026.3.13 不能用 1.0.6**：缺少 reply-payload、channel-reply-pipeline 模块
- 1.0.6 要求 openclaw ≥2026.4.5（compat.pluginApi: ^2026.4.5）
- 0.4.3 是扁平格式，不要求新 API，但 0.4.3 的 compat 字段没限制版本

### 扁平 vs instances 是 openclaw-nim 插件自己的约束，不是 openclaw 框架的
- 0.4.3 读 `cfg.channels.nim.appKey`
- 1.0.6 读 `cfg.channels.nim.instances[0].appKey`
