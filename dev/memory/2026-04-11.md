## openclaw-easy token 验证跳转问题修复 (最终)

### 问题根因（完整分析）
1. **setup.html** 的 `verifyTokenAndLogin` 函数逻辑有 bug：
   - `if (token && barCode)` 判断后 `return`，但代码继续执行了 API 调用
   - 当 token 为 undefined 时调用 `/api/verifyToken`，导致报错"token参数必传"

2. **三个文件跳转方式不一致**：
   - login.html/setup.html/index.html 使用 `window.location.search`
   - 但部分场景下 `window.location.search` 可能丢失参数

### 最终修复
- **login.html**: 第 98 行改为显式传参
- **setup.html**: 第 98 行改为显式传参，移除残留 API 调用
- **index.html**: 第 1677 行改为显式传参

### 统一跳转代码
```javascript
window.location.href = '/verify.html?token=' + encodeURIComponent(token) + '&barCode=' + encodeURIComponent(barCode);
```

### 页面职责（最终确认）
| 页面 | token+barCode | 无 token |
|------|---------------|----------|
| login.html | 跳转 verify.html | 显示登录表单 |
| setup.html | 跳转 verify.html | 显示设置表单 |
| index.html | 跳转 verify.html | 正常加载首页 |
| verify.html | 调用 /api/verifyToken | 跳转 login.html |

## 认证流程完整梳理（最终版）

### 流程图
```
用户访问 URL（带 token+barCode）
         │
         ▼
入口页面检测（login/setup/index）
         │
    if (token && barCode)?
    ├─ YES → 跳转 verify.html?token=xxx&barCode=xxx
    │         return; // 立即返回，不执行后续代码
    │
    └─ NO → 执行原有逻辑（显示表单/加载首页）
         │
         ▼
    verify.html
         │
    if (!token || !barCode)?
    ├─ YES → 跳转 login.html
    │
    └─ NO → POST /api/verifyToken
              ├─ 成功 → 跳转首页 (/)
              └─ 失败 → 显示错误信息
```

### 各页面职责（确认正确）

| 页面 | token+barCode 存在 | token+barCode 不存在 | API 调用 |
|------|-------------------|---------------------|---------|
| login.html | 跳转 verify.html | 显示登录表单 | ❌ 无 |
| setup.html | 跳转 verify.html | 显示设置表单 | ❌ 无 |
| index.html | 跳转 verify.html | 正常加载首页 | ❌ 无 |
| verify.html | 调用 /api/verifyToken | 跳转 login.html | ✅ 有 |

### 关键设计原则

1. **入口页面不调用验证 API** - login/setup/index 只做跳转
2. **立即返回** - `if (token && barCode)` 后立即 `return`
3. **显式传参** - `encodeURIComponent(token)` 而不是 `location.search`
4. **参数校验** - verify.html 先校验 `if (!token || !barCode)`

### 防止"token必传参数"错误的机制

| 机制 | 实现方式 |
|------|---------|
| 入口分离 | login/setup/index 不调用 API |
| 立即返回 | if 后 return 阻断后续执行 |
| 显式传参 | encodeURIComponent 传参 |
| 参数校验 | verify.html 先校验参数 |

### index.html 说明
- `index.html` 是首页 `/` 的实际文件（不是新加的）
- 静态路由: `express.static(public)` → 访问 `/` 返回 `public/index.html`
- APP 扫码后访问 `/?token=xxx&barCode=xxx` 时会跳转 verify.html
