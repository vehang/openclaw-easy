## openclaw-easy 项目

### 版本管理
- 新增 `version.json`：包含 `versionName`（如 "1.0.0"）和 `versionCode`（如 1000）
- 版本检查接口：`GET https://api.yun.tilldream.com/api/nas/fw/getNewVersionV2?platform=openclaw-easy&versionCode=1000`

### API 接口
- `GET /api/version` - 获取当前版本信息
- `GET /api/update/check` - 检查更新
- `POST /api/update` - 一键更新（无参数，自动检查下载安装重启）
- `POST /api/fix` - 执行 `openclaw doctor --fix` 修复运行环境

### 自动更新任务
- 启动后延迟 30 秒检查更新
- 每 6 小时定时检查更新
- 环境变量控制：`AUTO_UPDATE`, `AUTO_UPDATE_ON_START`, `AUTO_UPDATE_INTERVAL`, `AUTO_UPDATE_INSTALL`

### 更新回滚机制
- `docker/update.sh`：独立更新脚本，不依赖 Node.js 进程
- `docker/start-check.sh`：启动检查脚本，检测失败更新并自动回滚
- 标记文件：`/tmp/openclaw-easy-update-marker`
- 更新失败时保留标记文件，下次启动自动回滚

### 构建目录统一
- 所有构建脚本统一到 `docker/` 目录
- `docker/build.sh` - 主构建脚本
- `docker/Dockerfile.integrated` - 整合版镜像

### 其他修改
- 移除保存配置时的自动重启逻辑

- 所有 API 返回格式统一为 `{ code, msg, currentTime, data }`

### 更新回滚脚本修复（晚上）
修复的问题：
1. **标记文件过早删除**（严重）
   - 问题：update.sh 成功后立即删除标记文件
   - 后果：重启失败时无法检测到需要回滚
   - 修复：保留标记文件，由 start-check.sh 启动成功后删除

2. **set -e 导致意外退出**
   - 问题：任何命令失败就退出脚本
   - 修复：移除 set -e，使用显式错误处理

3. **子目录检测逻辑错误**
   - 问题：多个文件时 wc -l > 1，但可能是单目录
   - 修复：使用 ls -A 和 wc -w 正确检测

4. **缺少命令检查**
   - 问题：直接使用 wget，可能不存在
   - 修复：检查 wget/curl，优先使用可用的

5. **回滚失败未处理**
   - 问题：回滚失败仍显示成功
   - 修复：返回正确的退出码

6. **无限回滚风险**（严重）
   - 问题：连续失败会无限回滚
   - 修复：添加回滚计数文件，最多 3 次

新增功能：
- 启动成功后自动清理标记文件（10秒后检测）
- 回滚计数文件：`/tmp/openclaw-easy-rollback-count`
- 日志记录：`/var/log/supervisor/update.log`
- `update.sh clear-marker` 命令手动清理标记

### 完整更新流程
```
update.sh update <url>
    ↓
备份 + 写标记文件
    ↓
下载 → 解压 → 更新文件 → 更新依赖
    ↓
保留标记文件，返回
    ↓
restartEasy() 重启服务
    ├─ 成功 → start-check.sh 检测服务正常 → 删除标记
    └─ 失败 → 标记保留 → supervisord 重启 → start-check.sh 回滚
```
