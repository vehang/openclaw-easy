# 登录成功后首页数据刷新机制

## 功能概述

当用户在首页进行登录或注册成功后，系统会自动触发frequent-tools接口调用，刷新常用工具数据的展示。

## 实现原理

### 1. 事件驱动架构

系统采用事件驱动的方式来处理登录成功后的数据刷新：

- **登录成功事件**: `LOGIN_SUCCESS_HOME_REFRESH`
- **事件详情**: 包含登录类型（login/register）、时间戳和用户数据

### 2. 核心组件

#### LoginModal组件
- 检测登录成功后的当前页面路径
- 如果在首页（`/` 或 ``）则触发专门的首页刷新事件
- 使用统一的事件处理函数 `handleLoginSuccessEvent`

#### 首页（page.tsx）
- 监听 `LOGIN_SUCCESS_HOME_REFRESH` 事件
- 收到事件后执行两个操作：
  1. 刷新首页主要数据（工具分类数据）
  2. 延迟500ms后调用 `smartSync()` 刷新常用工具数据

#### useFrequentToolsSync Hook
- 提供 `smartSync()` 方法
- 自动判断是否需要恢复数据或同步数据
- 调用相应的API接口获取最新的常用工具数据

### 3. 数据流程

```
用户登录成功
    ↓
触发 LOGIN_SUCCESS_HOME_REFRESH 事件
    ↓
首页监听到事件
    ↓
1. 立即刷新首页数据（forceRefresh=true）
    ↓
2. 延迟500ms后调用smartSync()
    ↓
3. smartSync()根据情况调用：
   - recover-frequent-tools (如果需要恢复)
   - sync-frequent-tools (如果需要同步)
   - frequent-tools (获取聚合数据)
    ↓
4. 更新常用工具UI展示
```

### 4. 关键特性

- **智能判断**: 根据当前页面路径决定触发哪种刷新事件
- **防抖处理**: 避免短时间内重复刷新
- **延迟执行**: 确保登录状态已更新后再刷新常用工具
- **错误处理**: 包含完整的错误处理和日志记录

## 使用场景

1. **未登录用户首次登录**: 会触发数据恢复和同步
2. **已登录用户重新登录**: 会刷新最新的聚合数据
3. **新用户注册**: 会创建新的常用工具数据记录

## 配置项

- **延迟时间**: 500ms（可在page.tsx中调整）
- **防抖间隔**: 2秒（可在page.tsx中调整）
- **自动同步间隔**: 30秒（可在useFrequentToolsSync中调整）

## 日志输出

系统会输出详细的日志信息，便于调试：

```
🎉 首页检测到登录成功事件: login, 时间戳: 1696579200000
🔄 开始刷新常用工具数据...
✅ 常用工具数据刷新完成
```

## 注意事项

1. 该机制仅在首页生效
2. 书签页和收藏页有独立的事件处理机制
3. 其他页面使用通用的登录状态变化处理
4. 确保在组件卸载时正确清理事件监听器
