# 🔒 推送硬规则 — Notify Bot

> **铁律：只有最终的有效内容才推送给用户。系统执行过程、错误日志、调试信息一律不推。**

---

## 一、允许推送的内容（白名单）

只有以下 4 类消息可以推送给用户：

| # | 类型 | 时间 | 示例 |
|---|------|------|------|
| 1 | 🌤️ 天气早报卡片 | 每天 07:00 | 宜昌夷陵区天气 + 穿搭/出行建议 |
| 2 | 📰 科技资讯早报 | 每天 08:00 | AI/芯片/云计算相关新闻摘要 |
| 3 | 📰 科技资讯午报 | 每天 12:00 | 同上 |
| 4 | 📰 科技资讯晚报 | 每天 20:00 | 同上 |

**格式要求**：飞书交互卡片（interactive），包含 header + 分类 fields + 分隔线。

---

## 二、禁止推送的内容（黑名单）

以下内容**绝对不允许**推送给用户：

- ❌ Cron 任务执行结果（成功/失败/错误码）
- ❌ 脚本执行的 stdout / stderr
- ❌ API 调用日志（token 获取、HTTP 状态码等）
- ❌ "天气推送完成"、"资讯发送成功" 等确认消息
- ❌ 系统通知、心跳、状态报告
- ❌ 调试信息、错误堆栈
- ❌ GitHub 仓库监控（暂未正式启用，不推）
- ❌ git 备份结果
- ❌ 任何非白名单内容

**判断原则**：用户不需要知道"做了什么"，只需要看到"最终结果"。

---

## 三、任务执行规范

### 3.1 执行流程

```
cron 触发 → agent 执行任务 → 生成最终卡片 → 直接发送飞书卡片
                                        ↓
                            不走 announce/delivery
                            不回传执行结果
```

### 3.2 错误处理

- 执行失败时：**静默处理**，不推送错误信息
- 脚本不存在时：**用内联方式直接完成任务**（不报错、不推报错）
- API 超时时：静默重试 1 次，仍失败则跳过本次推送
- **永远不要把技术错误推给用户**

### 3.3 推送目标

- 天气早报 → 飞书私聊 `ou_7172afb8fa3c2f51bb02b6af097a4b27`
- 科技资讯 → 飞书私聊 `ou_7172afb8fa3c2f51bb02b6af097a4b27`

## 四、Cron 任务清单

| 任务 ID | 名称 | Cron | 状态 | 推送内容 |
|---------|------|------|------|----------|
| weather-morning | 天气早报 | `0 7 * * *` | ✅ | 天气卡片 |
| news-morning | 资讯早报 | `0 8 * * *` | ✅ | 资讯卡片 |
| news-noon | 资讯午报 | `0 12 * * *` | ✅ | 资讯卡片 |
| news-evening | 资讯晚报 | `0 20 * * *` | ⚠️ 上次失败 | 资讯卡片 |

### Cron delivery 配置原则

- `delivery.mode` 设为 `"silent"` 或不设 announce
- 由 agent 自行通过 `message` 工具发送最终卡片
- 不走系统 announce 通道（避免推送系统执行结果）
