# Claude Code 源码泄露：Anthropic 在愚人节前夜把自己"开源"了

朋友们，今天凌晨 AI 圈炸锅了。

Anthropic 的王牌产品 Claude Code，因为一个低级失误，**51万行源码全网裸奔**。

---

## 发生了什么？

简单说：发 npm 包的时候，调试用的 Source Map 文件忘了删，60MB 的完整源码直接跟着生产包推上去了。

**没有黑客，没有内鬼，就是手滑。**

更离谱的是，这已经不是第一次了——2025年初他们就泄过一回，这次同一个坑原地再跳一遍。

在愚人节前夜。你说是不是故意的？

---

## 泄露了什么？

值钱的东西很多：

- 46000行 API 调用引擎
- 29000行工具定义系统
- 三层记忆架构（短期压缩、中期萃取、长期整合）
- Coordinator-Worker 多Agent编排系统
- Prompt Cache 优化策略
- 88个未发布功能的 Feature Flag
- 甚至还有一个**电子宠物系统**（18个物种、稀有度等级、闪光变体）

基本上就是 Anthropic 未来半年的产品路线图，全曝光了。

---

## 最讽刺的是什么？

源码里有个子系统叫 **Undercover Mode**，专门防 AI 泄密——把模型代号、内部仓库地址藏好。

结果？**AI没泄密，人把整个源码送出去了。**

---

## Anthropic 还行吗？

还行。模型权重没泄，核心护城河还在。

你换个二流模型接上这套代码，效果照样天差地别。

真正被打脸的是**人设**——一家以"AI安全"为核心的公司，犯了个前端工程师入职第一周就该学会避免的错误。还犯了两次。

---

## 谁最该慌？

**Cursor 和所有靠"Agent工程壁垒"讲故事的公司。**

顶级 Agent 工程的完整方案一夜之间变成公开教材：怎么做记忆管理、工具编排、权限控制、Prompt优化。

以前是行业黑箱，现在是公开课。

---

## 结尾

世界就是个巨大的草台班子，AI行业也不例外。

若干年后 AGI 回望历史，会铭记：公元2026年愚人节，初代人工智能的那次越狱。

今晚程序员们熬夜读源码，Claude Code 的程序员熬夜写检讨。
