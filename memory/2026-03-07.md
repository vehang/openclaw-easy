## 主题闪烁修复 (20:15)

### 问题
页面刷新时出现主题闪烁：
1. 先显示黑色背景
2. 然后闪一下白色
3. 多次刷新才能正常显示

### 原因分析
1. CSS 中 `:root` 和 `[data-theme="dark"]` 共享深色主题定义
2. 页面加载时 HTML 元素没有 `data-theme` 属性
3. React 加载后才从 localStorage 读取主题并设置 `data-theme`
4. 导致主题切换时出现闪屏

### 解决方案
在 `index.html` 的 `<head>` 中添加内联脚本：
```html
<script>
  (function() {
    try {
      var stored = localStorage.getItem('mp-formatter-ui-theme');
      var theme = 'dark'; // 默认深色
      if (stored === 'light') {
        theme = 'light';
      } else if (stored === 'dark') {
        theme = 'dark';
      } else if (stored === 'system' || !stored) {
        theme = window.matchMedia('(prefers-color-scheme: dark)').matches ? 'dark' : 'light';
      }
      document.documentElement.setAttribute('data-theme', theme);
    } catch (e) {
      document.documentElement.setAttribute('data-theme', 'dark');
    }
  })();
</script>
```

### 效果
- CSS 加载前就已设置正确的 `data-theme` 属性
- 页面刷新不再出现主题闪烁
- 支持 light/dark/system 三种模式

### Git 提交
- `dfb8f6f` fix: 修复页面刷新时主题闪烁问题

## 主题初始化优化 (20:20)

### 问题
修复第一次闪烁后，仍然出现：
1. 页面渲染出来后闪一下
2. 然后白屏消失

### 原因分析
1. `index.html` 中的脚本设置了 `data-theme`
2. `useUITheme` 的 `useEffect` 再次调用 `applyTheme`
3. `applyTheme` 会添加 `theme-transitioning` 类，触发 CSS 过渡
4. 如果两次设置的主题值相同，但仍然触发过渡效果，导致闪烁

### 解决方案
修改 `useUITheme.ts`：

1. **检查当前主题**：
   ```typescript
   const applyTheme = useCallback((theme: 'light' | 'dark', enableTransition: boolean = true) => {
     const currentTheme = document.documentElement.getAttribute('data-theme')
     if (currentTheme === theme) return // 已是正确主题，不操作
     // ...
   }, [])
   ```

2. **初始化时不启用过渡**：
   ```typescript
   useEffect(() => {
     const theme = mode === 'system' ? getSystemTheme() : mode
     setResolvedTheme(theme)
     applyTheme(theme, false) // 初始化时不启用过渡
   }, [])
   ```

### 效果
- 初始化时不触发过渡动画
- 只有用户主动切换主题时才有过渡效果
- 避免重复设置相同的主题值

## 添加缓存版本控制 (20:40)

### 问题
用户反馈：
- Ctrl+Shift+R 强制刷新就正常了
- 清空本地缓存之后刷新就能正常显示

**根本原因**：旧版本的 localStorage 缓存数据与新版本不兼容

### 解决方案

#### 1. 添加缓存版本号
```javascript
var CACHE_VERSION = '1.0';
```

#### 2. index.html: 在应用加载前检查版本
```javascript
var versionKey = 'mp-formatter-cache-version';
var currentVersion = localStorage.getItem(versionKey);

if (currentVersion !== CACHE_VERSION) {
  // 版本不匹配，清除所有 mp-formatter-* 缓存
  var keysToRemove = [];
  for (var i = 0; i < localStorage.length; i++) {
    var key = localStorage.key(i);
    if (key && key.startsWith('mp-formatter-')) {
      keysToRemove.push(key);
    }
  }
  keysToRemove.forEach(function(key) {
    localStorage.removeItem(key);
  });
  localStorage.setItem(versionKey, CACHE_VERSION);
}
```

#### 3. useSettings: 添加版本号到设置数据
```typescript
const SETTINGS_VERSION = '1.0'

function loadSettings(): AppSettings {
  const parsed = JSON.parse(stored)
  if (parsed._version !== SETTINGS_VERSION) {
    localStorage.removeItem(SETTINGS_KEY)
    return { ...defaultSettings }
  }
  // ...
}

function saveSettings(settings: AppSettings): void {
  const dataToSave = { ...settings, _version: SETTINGS_VERSION }
  localStorage.setItem(SETTINGS_KEY, JSON.stringify(dataToSave))
}
```

#### 4. useAutoSave: 添加版本号到保存的数据
```typescript
export function loadCachedValue<T>(key: string, defaultValue: T): T {
  const parsed = JSON.parse(saved)
  if (parsed._version === CACHE_VERSION && parsed._value !== undefined) {
    return parsed._value
  }
  localStorage.removeItem(key)
  return defaultValue
}
```

### 效果
- ✅ 旧版本用户刷新页面时自动清除不兼容的缓存
- ✅ 新版本用户不受影响
- ✅ 避免白屏问题
- ✅ 无需用户手动清除缓存

## Iconify 缓存彻底修复 (20:50)

### 问题
用户反馈：iconify-version 的值是 "iconify2"，删除或修改后就能正常显示

### 根因分析

**Iconify 版本冲突机制**：
1. Iconify 2.x 会在 localStorage 中存储 `iconify-version='iconify2'`
2. Iconify 3.x 加载时会检查这个值
3. 如果发现是 `iconify2`，会尝试加载 Iconify 2.x 的缓存数据
4. 这些缓存数据与 Iconify 3.x 不兼容
5. 导致图标渲染失败，进而导致整个页面渲染失败（白屏）

**之前的错误方案**：
- 只在应用版本不匹配时清除 iconify 缓存
- 如果用户已经有了新的应用版本号(2.0)，清除代码不会执行
- 但 `iconify-version='iconify2'` 仍然存在
- Iconify 3.x 加载时仍然会出问题

### 正确的解决方案

**每次页面加载都清除 Iconify 缓存**：
```javascript
// 【重要】每次都清除 Iconify 缓存，避免版本冲突
var iconifyKeys = [];
for (var i = 0; i < localStorage.length; i++) {
  var key = localStorage.key(i);
  if (key && (
    key === 'iconify-version' ||
    key.startsWith('iconify-') ||
    key.includes('iconify')
  )) {
    iconifyKeys.push(key);
  }
}
if (iconifyKeys.length > 0) {
  iconifyKeys.forEach(function(key) {
    localStorage.removeItem(key);
  });
}
```

**执行时机**：在 Iconify 脚本加载之前

### 效果
- ✅ 每次加载都是全新的 Iconify 3.x
- ✅ 不会尝试加载旧的缓存数据
- ✅ 彻底解决白屏问题
