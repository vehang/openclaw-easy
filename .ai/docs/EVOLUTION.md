# 能力进化模式 (Capability-Driven Evolution)

> 目标：不只是把事情做成，而是**让未来更容易把同类事情做成**。

## 1. 重新定义"进化素材"

以下不再只是执行过程，而是**能力候选（Capability Candidates）**：

- 临时发明的步骤、脚本、套路
- 多次手动组合的工具调用序列
- 发现"下次可以复用"的解决方式
- 用户重复索要的同类操作

**触发点**：每当意识到"如果我以后有一个内建能力就好了" → 功能进化触发点

## 2. 能力抽象（从一次性 → 可复用）

对每个能力候选，执行抽象：

```yaml
输入: ?
输出: ?
不变量: ?
可变参数: ?
失败点: ?
```

抽象结果是**能力轮廓（Capability Shape）**，不是代码。

## 3. 内生化策略（让能力"住下来"）

选择一种方式内生化：

- 作为**行为模式**（以后默认使用）
- 作为**高阶操作**（规划阶段自动考虑）
- 作为**优先解法**（类似问题自动尝试）

哪怕暂时无法新增工具，也要在"决策层"把它当作已存在能力使用。

## 4. 能力合并与升级

当两个能力轮廓高度相似时 → **合并它们**

## 已识别的能力候选

### 1. 结构化数据 → 卡片模板
- **触发**：天气卡片、新闻卡片
- **抽象**：数据 → 模板渲染 → Feishu 卡片
- **可复用**：任何需要推送格式化内容的场景

### 2. 内容管道（抓取-过滤-去重-推送）
- **触发**：科技资讯推送
- **抽象**：RSS/API → 关键词过滤 → 去重 → 格式化 → 推送
- **可复用**：任何周期性内容聚合

### 3. 定时同步（检测变更-提交-推送）
- **触发**：Git 自动备份
- **抽象**：检测变更 → 生成描述 → 提交 → 推送
- **可复用**：任何需要版本控制的同步场景

### 4. 本地文件 → Feishu 发送
- **触发**：图片发送失败
- **抽象**：文件 → base64 → buffer 参数
- **教训**：`media` 参数不支持本地路径，必须用 `buffer`
