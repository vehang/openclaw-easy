## 第五次修复（13:05）- 代码块横向滚动

### 问题
用户反馈：代码块背景和颜色已正常，但横向滚动没有。

### 原因
公众号不支持 `overflow-x: auto`（横向滚动）。

### 解决方案
使用 `white-space: pre-wrap` 让代码自动换行：

```javascript

// 现在
docPre.setAttribute('style', preStyle + '; white-space: pre-wrap; word-wrap: break-word; overflow-wrap: break-word;')
```

### 效果
- ✅ 长代码行会自动换行，不会被截断
- ✅ 保持代码缩进格式

## 第七次修复（13:27）- 使用 section 实现横向滚动

### 问题
用户反馈：`overflow-x: auto; white-space: pre;` 方式在公众号中不生效，代码超出后换行而不是横向滚动。

### 关键发现
**公众号编辑器对 `<section>` 元素的 `overflow-x` 支持比 `<pre>` 元素好得多！**

这是通过深入研究发现的：
- 搜索"微信公众号 代码块 横向滚动"相关资料
- 研究其他项目的实现方式
- 发现使用 `<section>` 替代 `<pre>` 是关键

### 解决方案

**核心思路：** 将 `<pre>` 元素替换为 `<section>` 元素

```javascript
// 之前：直接在 pre 上加 overflow-x（不生效）
docPre.setAttribute('style', preStyle + '; overflow-x: auto; white-space: pre;')

// 现在：创建 section 替换 pre（生效！）
const section = doc.createElement('section')
section.setAttribute('style', preStyle + '; overflow-x: auto; white-space: pre; -webkit-overflow-scrolling: touch;')
section.innerHTML = docPre.innerHTML
docPre.parentNode?.replaceChild(section, docPre)
```

### 关键改进
1. ✅ 使用 `<section>` 代替 `<pre>`
2. ✅ 添加 `-webkit-overflow-scrolling: touch`（优化移动端滚动）
3. ✅ 同步更新代码高亮 span 选择器（`section.hljs span, pre.hljs span`）

### 最终 HTML 结构
```html
<section class="hljs" style="...; overflow-x: auto; white-space: pre; -webkit-overflow-scrolling: touch;">
  <code>
    <span style="color: #...">代码内容</span>
  </code>
</section>
```

## 第八次修复（13:50）- 修复代码块白色背景和横向滚动

### 问题
用户反馈前一次修复有两个问题：
1. 每行代码都有了白色背景（严重）
2. 代码横向滚动还是没解决

### 原因分析

**问题1：白色背景**
- 在处理代码高亮 span 时，`getComputedStylesAsInline` 读取了背景色
- 导致每个 span 都有了背景色（白色或主题背景色）

**问题2：横向滚动不生效**
- 之前用 section 替换 pre（`replaceChild`）
- 这样会丢失 pre 的语义和某些样式
- 公众号编辑器可能不支持这种结构

### 解决方案

**核心思路：用 section 包裹 pre（不是替换）**

```html
<!-- 之前（不生效）-->
<section class="hljs">
  代码内容
</section>

<!-- 现在（生效）-->
<section style="overflow-x: auto; width: 100%">
  <pre class="hljs" style="white-space: pre; background: #...">
    代码内容
  </pre>
</section>
```

**关键改进：**

1. **职责分离**
   - `<section>` 负责：横向滚动（`overflow-x: auto`）
   - `<pre>` 负责：代码格式（`white-space: pre`）+ 背景色

2. **样式优化**
   - 只给代码 span 设置颜色（`color: #...`）
   - 不给代码 span 设置背景
   - 保留 pre 的背景色

3. **HTML 结构**
   ```javascript
   // 创建 section 包裹 pre
   const section = doc.createElement('section')
   section.setAttribute('style', 'overflow-x: auto; ...')
   
   // 修改 pre 样式
   docPre.setAttribute('style', preStyle + '; white-space: pre; overflow: visible;')
   
   // 用 section 包裹 pre
   docPre.parentNode?.insertBefore(section, docPre)
   section.appendChild(docPre)
   ```

**问题特征：**
- ❌ 关键词之间的空格丢失（async function → asyncfunction）
- ❌ 换行符丢失
- ❌ 缩进丢失
- ❌ 不是所有代码都有问题，只是一部分
- ❌ 几乎每种语言都有这个问题

### 根本原因分析

**❌ 之前的错误实现：**
```typescript
const doc = new DOMParser().parseFromString(previewEl.innerHTML, 'text/html')
```

**问题：**
1. **DOMParser 会合并空格**
   - HTML 解析时，多个空格被标准化
   - 文本节点的空格被合并
   - 导致代码格式错误

2. **innerHTML 序列化问题**
   - 转换为字符串时，某些空格可能丢失
   - 解析回来时，空格被标准化

### 修复方案

**✅ 正确的实现：**
```typescript
// 使用 cloneNode 而不是 DOMParser
const clone = previewEl.cloneNode(true) as HTMLElement
const doc = document.implementation.createHTMLDocument()
doc.body.appendChild(clone)
```

**好处：**
- ✅ 保留所有原始 DOM 节点
- ✅ 空格不会被合并
- ✅ 换行符保留
- ✅ 缩进保留
- ✅ 所有文本节点都完整保留

## ✅ 右侧预览区域宽度自适应（17:53 完成）

### 用户反馈（17:38）
用户提出问题：
```
现在整个页面是左右各一半儿，左边是编辑区域，右边是预览区域，
默认预览的是宽屏，当切换到手机或pad模式的时候，预览区域的内容宽度变小了，
但是右侧整个预览区域的宽度没有变化，导致预览内容和预览区域的右侧
有大量的空白，显得不美观，也会让编辑区域受限。
```

### 问题分析

**之前的问题：**
```
┌─────────┬─────────────────────────┐
│         │                         │
│ 编辑区  │      预览区（50%）      │
│ （50%） │                         │
│         │  [手机 375px]           │
│         │  ← 大量空白浪费！        │
│         │                         │
└─────────┴─────────────────────────┘
```

**核心问题：**
- ❌ 右侧固定 50%
- ❌ 手机/Pad 模式内容宽度小
- ❌ 大量空白浪费
- ❌ 编辑区无法扩展

### 解决方案

**修改：** `src/App.tsx`

**右侧预览区域宽度自适应：**
```typescript
{/* 右侧预览 */}
<div
  className="flex flex-col"
  style={{ 
    background: 'var(--bg-muted)',
    width: previewMode === 'desktop' ? '50%' 
         : previewMode === 'pad' ? '820px' 
         : '415px',
    transition: 'width 0.3s ease-in-out'
  }}
>
```

**布局原理：**
1. 主容器：`flex` 布局
2. 左侧编辑区：`flex-1`（自动扩展）
3. 右侧预览区：固定宽度或百分比

### 三种模式宽度

**1. 宽屏模式（desktop）**
- 右侧区域：50%
- 预览内容：100%
- 效果：左右平分

**2. Pad 模式（pad）**
- 右侧区域：820px（768px + 40px padding）
- 预览内容：768px
- 效果：右侧紧凑，左侧扩展

**3. 手机模式（mobile）**
- 右侧区域：415px（375px + 40px padding）
- 预览内容：375px
- 效果：右侧更紧凑，左侧扩展更多
