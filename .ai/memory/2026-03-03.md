## Docker 升级配置问题

### 挂载目录分析
- ✅ `/root/.openclaw/workspace` - 已挂载（工作区）
- ❌ `/home/data/docker/openclaw/data` - 不存在

### 配置文件位置（不在 workspace 目录）
- `/root/.openclaw/openclaw.json` - 主配置（包含飞书账号、模型配置）
- `/root/.openclaw/credentials/` - 凭证缓存
- `/root/.openclaw/feishu/` - 飞书相关

### 升级容器后配置丢失风险
**问题**：如果 Docker 容器只挂载 `workspace` 目录，升级容器时 `/root/.openclaw/` 下的配置会丢失

**解决方案**：
1. 挂载整个 `/root/.openclaw/` 目录
2. 或者升级前备份配置到 `workspace/backup-configs/`（已实现）
3. 使用 `openclaw.json.template` + 手动填写敏感信息恢复

### Docker Compose 配置最佳实践

```yaml
volumes:
  - ${openclaw_data_dir}:/root/.openclaw  # 挂载整个配置目录
```

**注意**：不要排除 `extensions/` 子目录
```yaml
# ❌ 错误示例：排除 extensions 会导致升级后插件不更新
volumes:
  - /root/.openclaw/extensions  # 匿名卷，会保留旧版本
```

**正确做法**：让 extensions 随镜像更新
```bash
# 升级时删除匿名卷
docker-compose down -v && docker-compose up -d
```

## 配对问题

### 升级后配对丢失
- **原因**：版本不一致导致配对数据格式变化
- **现象**：`Config was last written by a newer OpenClaw (2026.3.2); current version is 2026.3.1`
- **解决**：
  1. 重新批准：`openclaw pairing approve feishu J6JUWUUK`
  2. 重启容器让版本一致：`docker-compose down -v && docker-compose up -d`

### 配对数据存储位置
- `/root/.openclaw/credentials/feishu-allowFrom.json` - 全局允许列表
- `/root/.openclaw/credentials/feishu-{account}-allowFrom.json` - 按账号的允许列表
- `/root/.openclaw/credentials/feishu-pairing.json` - 待批准的配对请求
- **注意**：这些文件在 `credentials/` 目录，不在 `extensions/`，升级容器会保留

### 升级后需要重新配对的原因
1. **版本不一致**：运行版本 2026.3.1 ≠ 配置版本 2026.3.2
2. **npm 升级后未重启容器**：新版本配置格式无法被旧版本正确读取
3. **正确流程**：`npm install -g openclaw@latest` → `docker restart <容器名>`

**推荐升级方式**：
```bash
docker-compose down -v
docker pull openclaw/openclaw:latest
docker-compose up -d
```
