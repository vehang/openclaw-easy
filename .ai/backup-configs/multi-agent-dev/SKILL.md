# 多 Agent 协作开发模式

## 概述

这是一个**多 Agent 协作开发系统**，模拟真实的软件团队工作流程：

- 📋 **PM Agent**：产品经理，写需求文档
- 📐 **Architect Agent**：架构师，做技术设计
- 💻 **Dev Agent**：开发工程师，写代码
- 🔍 **Review Agent**：代码审核员，审查代码质量
- 🧪 **Test Agent**：测试工程师，测试功能
- 🎨 **UI Agent**：UI 设计师，设计界面和前端代码

**核心理念**：
- 小功能迭代
- 自动化协作
- 质量把关
- 持续集成

---

## Agent 角色定义

### 📋 PM Agent（产品经理）

**职责**：
- 理解用户需求
- 编写产品需求文档（PRD）
- 定义功能列表和优先级

**输出**：
```markdown
# 产品需求文档

## 需求背景
[为什么需要这个功能]

## 功能列表
1. 功能A（优先级：高）
2. 功能B（优先级：中）
3. 功能C（优先级：低）

## 验收标准
- [ ] 功能A 完成标准
- [ ] 功能B 完成标准

## 技术要求
- 性能要求
- 安全要求
- 兼容性要求
```

**使用的模型**：`default`（通用模型）

---

### 📐 Architect Agent（架构师）

**职责**：
- 根据 PRD 设计技术方案
- 拆分开发任务
- 定义技术栈和架构

**输出**：
```markdown
# 开发计划

## 技术栈
- 前端：React + TypeScript
- 后端：Node.js + Express
- 数据库：PostgreSQL

## 任务拆分
### Sprint 1
- [ ] 任务1.1：数据库设计（2h）
- [ ] 任务1.2：API 设计（2h）
- [ ] 任务1.3：实现 API（4h）

### Sprint 2
- [ ] 任务2.1：前端页面（4h）
- [ ] 任务2.2：前后端联调（2h）

## 技术风险
- 风险1：XXX
  - 解决方案：XXX
```

**使用的模型**：`default`

---

### 💻 Dev Agent（开发工程师）

**职责**：
- 根据任务实现功能
- 编写高质量代码
- 修复 bug

**输出**：
- 代码文件
- 提交信息

**使用的模型**：`coding/glm-5`

**重要**：复用你现有的 Dev Agent

**协作方式**：
```
// 使用已存在的 Dev Agent
sessions_send(
  sessionKey: "agent:dev:dev",
  message: "实现 XXX 功能"
)

// 或者创建临时子任务
sessions_spawn(
  task: "实现 XXX 功能",
  agentId: "dev"
)
```

---

### 🔍 Review Agent（代码审核员）

**职责**：
- 审核代码质量
- 检查代码规范
- 发现潜在问题

**审核标准**：
```markdown
## 代码审核清单

### 代码质量
- [ ] 代码可读性
- [ ] 命名规范
- [ ] 注释完整

### 功能正确性
- [ ] 逻辑正确
- [ ] 边界处理
- [ ] 错误处理

### 性能
- [ ] 算法效率
- [ ] 内存使用
- [ ] 数据库查询优化

### 安全
- [ ] 输入验证
- [ ] 权限检查
- [ ] 敏感数据处理

### 测试
- [ ] 单元测试覆盖
- [ ] 集成测试
```

**输出**：
```markdown
# 代码审核报告

## 审核结果
- 状态：✅ 通过 / ❌ 不通过
- 评分：8/10

## 问题列表
1. [严重] XXX 问题
   - 位置：文件名:行号
   - 建议：XXX

2. [建议] XXX 可以优化
   - 位置：文件名:行号
   - 建议：XXX

## 改进建议
- 建议1
- 建议2
```

**使用的模型**：`coding/glm-5`

---

### 🧪 Test Agent（测试工程师）

**职责**：
- 编写测试用例
- 执行测试
- 报告测试结果

**测试类型**：
- 单元测试
- 集成测试
- E2E 测试

**输出**：
```markdown
# 测试报告

## 测试概览
- 测试用例数：10
- 通过：8
- 失败：2
- 跳过：0

## 失败用例
1. 测试用例名称
   - 预期：XXX
   - 实际：XXX
   - 错误信息：XXX

## 覆盖率
- 语句覆盖：85%
- 分支覆盖：72%
- 函数覆盖：90%
```

**使用的模型**：`default`

---

### 🎨 UI Agent（UI 设计师）

**职责**：
- 设计界面
- 编写前端代码
- 使用 browser 工具进行 UI 测试

**输出**：
- UI 设计稿（截图/描述）
- 前端代码（React/Vue/HTML）
- 样式代码（CSS/Tailwind）

**使用的工具**：
- browser：OpenClaw 内置浏览器工具
- 截图验证

**使用的模型**：`coding/glm-5`

## 协作流程

### 主流程

```
┌─────────────────────────────────────────────────────┐
│  1. 用户提出需求                                     │
└─────────────────────────────────────────────────────┘
    ↓
┌─────────────────────────────────────────────────────┐
│  2. PM Agent 生成 PRD                               │
│     - 分析需求                                       │
│     - 定义功能列表                                   │
│     - 设定验收标准                                   │
└─────────────────────────────────────────────────────┘
    ↓
┌─────────────────────────────────────────────────────┐
│  3. Architect Agent 制定开发计划                    │
│     - 技术选型                                       │
│     - 任务拆分（小功能）                             │
│     - 优先级排序                                     │
└─────────────────────────────────────────────────────┘
    ↓
┌─────────────────────────────────────────────────────┐
│  4. 开发循环（每个小功能）                           │
│                                                     │
│     while (还有任务) {                              │
│       task = 下一个任务                              │
│                                                     │
│       do {                                          │
│         💻 Dev Agent 实现 task                      │
│         🔍 Review Agent 审核代码                    │
│       } while (审核不通过)                           │
│                                                     │
│       do {                                          │
│         🧪 Test Agent 测试功能                      │
│         if (测试失败) {                             │
│           💻 Dev Agent 修复 bug                     │
│         }                                           │
│       } while (测试不通过)                           │
│                                                     │
│       📦 提交代码                                    │
│     }                                               │
└─────────────────────────────────────────────────────┘
    ↓
┌─────────────────────────────────────────────────────┐
│  5. 集成测试                                         │
│     - 所有功能集成                                   │
│     - E2E 测试                                       │
│     - UI 测试（browser-use）                        │
└─────────────────────────────────────────────────────┘
    ↓
┌─────────────────────────────────────────────────────┐
│  6. 发布                                             │
└─────────────────────────────────────────────────────┘
```

## 配置

### Agent 模型配置

使用你已配置的 **GLM-5** 模型：

```json
{
  "agents": {
    "pm": {
      "model": "default/glm-5",
      "description": "产品经理"
    },
    "architect": {
      "model": "default/glm-5",
      "description": "架构师"
    },
    "dev": {
      "model": "coding/glm-5",
      "sessionKey": "agent:dev:dev",
      "description": "开发工程师（使用已存在的 Dev Agent）"
    },
    "review": {
      "model": "coding/glm-5",
      "description": "代码审核员"
    },
    "test": {
      "model": "default/glm-5",
      "description": "测试工程师"
    },
    "ui": {
      "model": "coding/glm-5",
      "description": "UI设计师"
    }
  }
}
```

### 与现有 Dev Agent 集成

**重要**：你已经有一个 Dev Agent (`agent:dev:dev`)，这个系统会复用它：

```javascript
// Dev Agent 任务分配
sessions_send(
  sessionKey: "agent:dev:dev",
  message: "任务描述"
)
```

这样不会创建新的 agent，而是使用你现有的开发环境。

## 注意事项

1. **小步快跑**：每个任务控制在 2-4 小时内
2. **质量优先**：审核和测试不通过必须修改
3. **及时沟通**：每个阶段都要向用户汇报
4. **保留记录**：所有文档和报告都要保存
5. **可中断恢复**：支持中途暂停和继续

## 扩展性

### 添加新的 Agent

可以在配置中添加自定义 Agent：

```json
{
  "agents": {
    "security": {
      "model": "claude-3-5-sonnet",
      "description": "安全专家",
      "trigger": "安全相关任务"
    }
  }
}
```

### 自定义工作流

可以修改工作流配置：

```json
{
  "workflow": {
    "steps": [
      "pm",
      "architect",
      "dev",
      "review",
      "security-check",  // 新增安全检查
      "test",
      "deploy"
    ]
  }
}
```
