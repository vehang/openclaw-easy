# 多 Agent 协作开发模式 - 快速开始

## 🚀 立即使用

### 方式一：直接对话

```
用多agent模式开发一个待办事项应用
```

### 方式二：指定功能

```
用多agent模式开发一个用户登录功能
需求：支持邮箱登录、手机登录、第三方登录
```

### 方式三：从 PRD 开始

```
这是一个PRD：[粘贴PRD内容]
用多agent模式实现它
```

---

## 📋 工作流程

当你发起一个开发任务后，系统会自动：

1. **📋 PM Agent** → 生成 PRD
2. **📐 Architect Agent** → 制定开发计划
3. **💻 Dev Agent** → 实现功能
4. **🔍 Review Agent** → 审核代码
5. **🧪 Test Agent** → 测试功能
6. **📦 提交代码**

每个环节如果发现问题，会自动返回上一个环节修改。

## 💡 使用技巧

### 1. 明确需求

```
❌ 不好的需求：
开发一个系统

✅ 好的需求：
开发一个用户登录系统
- 支持邮箱登录
- 支持手机登录
- 支持微信登录
- 需要验证码功能
```

### 2. 分阶段开发

```
第一阶段：核心功能
第二阶段：优化体验
第三阶段：扩展功能
```

### 3. 及时反馈

```
查看开发进度
查看最新的审核报告
这个功能不对，需要修改
```

## 📂 生成的文件

```
~/.openclaw/workspace/projects/{项目名}/
├── docs/
│   ├── prd.md           # 产品需求文档
│   ├── plan.md          # 开发计划
│   ├── reviews/         # 审核报告
│   │   ├── review-001.md
│   │   └── review-002.md
│   └── tests/           # 测试报告
│       ├── test-001.md
│       └── test-002.md
├── src/                 # 源代码
│   ├── backend/
│   └── frontend/
└── .multi-agent/        # 协作状态
    ├── progress.json    # 进度
    ├── tasks.json       # 任务列表
    └── sessions.json    # Agent 会话
```

## 🤖 Agent 介绍

| Agent | 职责 | 何时介入 |
|-------|------|----------|
| 📋 PM | 写PRD | 需求分析阶段 |
| 📐 Architect | 做计划 | 设计阶段 |
| 💻 Dev | 写代码 | 开发阶段 |
| 🔍 Review | 审核代码 | 代码完成后 |
| 🧪 Test | 测试功能 | 审核通过后 |
| 🎨 UI | 设计界面 | 前端开发时 |
