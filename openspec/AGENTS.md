# OpenSpec AI 助手指令

你是这个项目的 AI 助手。请遵循以下规范工作流程。

## 工作流程

### 1. 创建变更提案
当用户要求添加新功能或修改现有功能时：
```
/openspec:proposal <功能描述>
```

这会创建 `openspec/changes/<功能名>/` 目录，包含：
- `proposal.md` - 提案说明
- `tasks.md` - 任务清单
- `specs/` - 规范文件

### 2. 实施变更
```
/openspec:apply
```
按任务清单逐项实施。

### 3. 归档变更
```
/openspec:archive
```
完成后归档变更，更新主规范。

## 规范文件格式

### Requirement 格式
```markdown
## ADDED Requirements

### Requirement: <需求名称>
系统 MUST <需求描述>

#### Scenario: <场景名称>
- **GIVEN** <前置条件>
- **WHEN** <触发条件>
- **THEN** <预期结果>
```

### Tasks 格式
```markdown
# 实施任务

- [ ] 1.1 <任务描述>
- [ ] 1.2 <任务描述>
```

## 项目特定规范

### 主题系统
- 使用 CSS 变量（`--theme-*`）
- 支持 18 个预览主题
- 支持 6 个代码主题

### 样式规范
- 禁止蓝紫色渐变
- 使用温暖配色
- 支持深色/浅色模式

### 组件规范
- 使用函数式组件
- 使用 Hooks 管理状态
- 遵循 React 最佳实践

## 注意事项

1. 修改代码前先创建变更提案
2. 保持任务清单更新
3. 完成后归档变更
4. 遵循现有代码风格
