# 提案：修复公众号代码粘贴格式丢失

## Why（为什么）

用户从微信公众号复制富文本内容，粘贴到编辑器后，原本格式正常的代码块变成了单行显示，所有换行和格式都丢失了。

这个问题严重影响用户体验，特别是对于需要处理技术内容的用户。

## What（做什么）

1. 分析公众号富文本的 HTML 结构
2. 检查当前粘贴处理逻辑
3. 识别代码块的格式丢失原因
4. 实现正确的富文本粘贴处理

## Impact（影响）

- 影响文件：
  - `src/components/CodeMirrorEditor.tsx`（粘贴处理）
  - `src/utils/markdown.ts`（转换逻辑）
- 不影响现有功能
- 提升用户体验

## 根因分析（待确认）

可能的原因：
1. 粘贴时只获取了 `text/plain`，忽略了 `text/html`
2. HTML 转换 Markdown 时，`<pre>` 和 `<code>` 标签处理不当
3. 公众号代码块使用特殊样式，没有被正确识别

## 解决方案（待确认）

1. 优先读取剪贴板的 `text/html` 格式
2. 增强 HTML → Markdown 转换逻辑
3. 特殊处理公众号的代码块格式
